## Steps Component

Perfect for tutorials and step-by-step guides.

<Steps>
  <Step>
    ### Install Dependencies
    
    First, install the required packages:
    ```bash
    npm install fumadocs-ui fumadocs-core
    ```
  </Step>
  
  <Step>
    ### Configure Your Project
    
    Create a `source.config.ts` file in your project root:
    ```typescript
    import { defineDocs } from 'fumadocs-mdx/config';
    
    export const { docs, meta } = defineDocs({
      dir: 'content/docs',
    });
    ```
  </Step>
  
  <Step>
    ### Create Your First Page
    
    Add your first documentation page at `content/docs/index.mdx`:
    ```mdx
    ---
    title: Welcome
    description: Getting started with Fumadocs
    ---
    
    # Welcome to My Docs
    ```
  </Step>
  
  <Step>
    ### Start Development Server
    
    Run your development server and see your documentation:
    ```bash
    npm run dev
    ```
  </Step>
</Steps>

## Accordion Component

Expandable sections for organizing detailed information.

<Accordions>
  <Accordion title="What is Fumadocs?">
    Fumadocs is a documentation framework for Next.js. It provides a complete solution for building beautiful documentation sites with MDX, full-text search, and internationalization support.
  </Accordion>
  
  <Accordion title="Why should I use Fumadocs?">
    - **Fast**: Built on Next.js with static generation
    - **Beautiful**: Modern UI with dark mode support
    - **Flexible**: Customizable themes and layouts
    - **Developer-friendly**: Write in MDX with hot reload
    - **Search**: Built-in full-text search
    - **Type-safe**: Full TypeScript support
  </Accordion>
  
  <Accordion title="How do I customize the theme?">
    You can customize Fumadocs by:
    1. Modifying CSS variables in your global styles
    2. Creating custom components
    3. Using the theme configuration options
    4. Overriding default components with your own
  </Accordion>
  
  <Accordion title="Can I use custom MDX components?">
    Yes! You can register custom MDX components in your page configuration. Simply add them to the components object when rendering MDX content.
  </Accordion>

## Code Blocks

Fumadocs provides enhanced code blocks with syntax highlighting.

```typescript title="example.ts"
interface User {
  id: string;
  name: string;
  email: string;
  role: 'admin' | 'user';
}

function greetUser(user: User): string {
  return `Hello, ${user.name}!`;
}

// Usage
const user: User = {
  id: '123',
  name: 'John Doe',
  email: 'john@example.com',
  role: 'user'
};

console.log(greetUser(user));
```

### With Line Highlighting

```jsx {3-5,8} title="component.jsx"
function Counter() {
  const [count, setCount] = useState(0);
  // These lines are highlighted
  const increment = () => setCount(count + 1);
  const decrement = () => setCount(count - 1);
  
  return (
    <div> {/* This line is also highlighted */}
      <button onClick={decrement}>-</button>
      <span>{count}</span>
      <button onClick={increment}>+</button>
    </div>
  );
}
```
