## Callout Component

Callouts are used to highlight important information. They support different types and can include titles.

### Basic Callouts

```mdx
<Callout>
  This is a default callout with no type specified.
</Callout>
```

### Callout Types

- **Info callout**: Use for supplementary information that helps users understand the content better.
- **Tip callout**: Share helpful tips and best practices. Great for optimization suggestions.
- **Warning callout**: Alert users to potential issues or important considerations.
- **Error callout**: Highlight critical problems or breaking changes that require immediate attention.

```mdx
<Callout type="info">Info message</Callout>
<Callout type="tip">Helpful tip</Callout>
<Callout type="warning">Warning message</Callout>
<Callout type="error">Error message</Callout>
```

### Callouts with Titles

```mdx
<Callout type="info" title="Custom Title">
  Callout content with a custom title
</Callout>
```

### When to Use Callouts

- **Info**: API details, additional context, feature explanations
- **Tip**: Performance optimizations, shortcuts, best practices
- **Warning**: Deprecations, security concerns, compatibility issues
- **Error**: Breaking changes, critical bugs, data loss risks

## Code Blocks

Fumadocs provides enhanced code blocks with syntax highlighting, line numbers, and copy functionality.

### Basic Code Block

```javascript
// Basic JavaScript function
function calculateTotal(items) {
  return items.reduce((sum, item) => sum + item.price, 0);
}

const items = [
  { name: 'Apple', price: 1.99 },
  { name: 'Banana', price: 0.99 },
  { name: 'Orange', price: 2.49 }
];

console.log(`Total: $${calculateTotal(items).toFixed(2)}`);
```

### Code Block with Title

```typescript title="user.types.ts"
export interface User {
  id: string;
  name: string;
  email: string;
  role: 'admin' | 'user' | 'guest';
  createdAt: Date;
  updatedAt: Date;
}

export interface CreateUserDto {
  name: string;
  email: string;
  password: string;
  role?: User['role'];
}
```

### Code Block with Line Highlighting

```jsx {3-5,8}
function MyComponent() {
  const [count, setCount] = useState(0);
  // These lines are highlighted
  const increment = () => setCount(count + 1);
  const decrement = () => setCount(count - 1);
  
  return (
    <div> {/* This line is also highlighted */}
      <button onClick={decrement}>-</button>
      <span>{count}</span>
      <button onClick={increment}>+</button>
    </div>
  );
}
```

### When to Use Code Blocks

- API examples
- Configuration snippets
- Installation commands
- Code tutorials
- Error examples
- Query examples

## Dark Mode Behavior

All Fumadocs components automatically adapt to dark mode. Here's how different components behave:

### Component-Specific Dark Mode

1. **Callouts**: Background colors adjust while maintaining contrast
2. **Code Blocks**: Syntax highlighting themes switch automatically
3. **Cards**: Borders become more prominent in dark mode
4. **Tables**: Alternating row colors adapt to the theme
5. **Accordions**: Hover states adjust for visibility

### Testing Dark Mode

1. Toggle your system theme or use the theme switcher
2. Verify all text remains readable
3. Check that interactive elements are clearly visible
4. Ensure code syntax highlighting works correctly

## Best Practices

### Component Selection Guide

- **Use Callouts When**: Highlighting warnings or tips, adding context to complex topics, drawing attention to important info, providing quick takeaways
- **Use Cards When**: Creating feature grids, building navigation menus, showcasing options, making content scannable
- **Use Tabs When**: Showing platform variations, organizing code examples, presenting alternatives, reducing page length
- **Use Steps When**: Writing tutorials, documenting processes, creating guides, showing sequences

### Accessibility Considerations

All Fumadocs components are built with accessibility in mind:
- Proper ARIA labels
- Keyboard navigation support
- Screen reader compatibility
- High contrast ratios

### Performance Tips

1. **Lazy Loading**: Large code blocks and images are lazy-loaded
2. **Component Bundling**: Only import components you use
3. **Static Generation**: MDX content is pre-rendered for fast loading
4. **Responsive Design**: All components work on mobile devices
