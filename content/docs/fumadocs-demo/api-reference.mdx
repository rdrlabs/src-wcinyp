## REST API Example

<Tabs items={['cURL', 'JavaScript', 'Python', 'Go']}>
  <Tab value="cURL">
    ```bash
    curl -X GET https://api.example.com/v1/users \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -H "Content-Type: application/json"
    ```
  </Tab>
  <Tab value="JavaScript">
    ```javascript
    const response = await fetch('https://api.example.com/v1/users', {
      method: 'GET',
      headers: {
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
      }
    });
    const data = await response.json();
    ```
  </Tab>
  <Tab value="Python">
    ```python
    import requests
    
    response = requests.get(
        'https://api.example.com/v1/users',
        headers={
            'Authorization': 'Bearer YOUR_API_KEY',
            'Content-Type': 'application/json'
        }
    )
    data = response.json()
    ```
  </Tab>
  <Tab value="Go">
    ```go
    req, _ := http.NewRequest("GET", "https://api.example.com/v1/users", nil)
    req.Header.Add("Authorization", "Bearer YOUR_API_KEY")
    req.Header.Add("Content-Type", "application/json")
    
    res, _ := http.DefaultClient.Do(req)
    defer res.Body.Close()
    ```
  </Tab>
</Tabs>

## API Response Format

<Callout type="info">
  All API responses follow a consistent JSON structure with metadata and data fields.
</Callout>

### Success Response

```json
{
  "success": true,
  "data": {
    "users": [
      {
        "id": "123",
        "name": "John Doe",
        "email": "john@example.com",
        "created_at": "2024-01-15T10:00:00Z"
      }
    ]
  },
  "metadata": {
    "page": 1,
    "limit": 20,
    "total": 100
  }
}
```

### Error Response

```json
{
  "success": false,
  "error": {
    "code": "USER_NOT_FOUND",
    "message": "The requested user does not exist",
    "details": {
      "user_id": "456"
    }
  }
}
```

## Authentication

<Steps>
  <Step>
    ### Obtain API Key
    
    Register for an account and generate your API key from the dashboard:
    ```
    https://dashboard.example.com/api-keys
    ```
  </Step>
  
  <Step>
    ### Include in Requests
    
    Add your API key to the Authorization header:
    ```
    Authorization: Bearer YOUR_API_KEY
    ```
  </Step>
  
  <Step>
    ### Handle Token Expiry
    
    API keys expire after 90 days. Implement token refresh logic:
    ```javascript
    if (response.status === 401) {
      await refreshApiKey();
      // Retry request
    }
    ```
  </Step>
</Steps>

## Rate Limiting

<Banner>
  API requests are limited to 1000 requests per hour per API key
</Banner>

### Rate Limit Headers

| Header | Description | Example |
|--------|-------------|---------|
| `X-RateLimit-Limit` | Maximum requests per hour | `1000` |
| `X-RateLimit-Remaining` | Requests remaining | `950` |
| `X-RateLimit-Reset` | Reset timestamp (Unix) | `1642291200` |

## Best Practices

<Steps>
  <Step>
    ### Use Pagination
    
    Always paginate list endpoints to improve performance:
    ```
    GET /users?page=1&limit=20
    ```
  </Step>
  
  <Step>
    ### Include Idempotency Keys
    
    For POST requests, include an idempotency key:
    ```
    Idempotency-Key: unique-request-id
    ```
  </Step>
  
  <Step>
    ### Handle Errors Gracefully
    
    Implement exponential backoff for retries:
    ```javascript
    const backoff = Math.min(1000 * 2 ** attempt, 30000);
    await sleep(backoff);
    ```
  </Step>
</Steps>
